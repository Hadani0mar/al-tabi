# خطة تحسين ذاكرة الشات عبر Supabase

هذا الملف يوثق خطة وتنفيذ ذاكرة الشات الجديدة في تطبيق التقارير.

## الهدف

- تقليل التوكن المرسل إلى النموذج.
- جعل الوكيل يتعرف بسرعة على التقارير والأنماط السابقة.
- حفظ المحادثات كجلسات ورسائل منفصلة بدلا من حفظ كل المحادثة في حقل JSON واحد.
- ربط كل تقرير مولد برقم واضح يمكن طلبه لاحقا من الواجهة أو Telegram.

## مشروع Supabase

المشروع المستخدم:

```text
delivery-app
nsgmhijtaaenpqxxgjds
```

## الجداول الجديدة

```text
chat_sessions
chat_messages
report_artifacts
```

الجداول الجديدة مفعلة عليها RLS من البداية. الوصول المباشر ممنوع بسياسات افتراضية، والتطبيق يتعامل معها عبر RPCs تستخدم بصمة access token.

## الجداول القديمة

الجدول القديم يبقى موجودا كاحتياط انتقالي:

```text
user_chats
```

لا يحذف ضمن هذه الخطة. يتم استخدامه للقراءة الاحتياطية أو الترحيل فقط.

## دوال RPC

```text
upsert_chat_session
append_chat_message
get_chat_sessions
get_session_messages
delete_chat_session
get_session_context
get_failing_intents
migrate_user_chats_to_messages
save_report_artifact
get_report_artifact_by_number
```

## سلوك التطبيق المطلوب

- الواجهة ترسل معرف الجلسة مع كل طلب AI.
- الخلفية تحفظ رسالة المستخدم قبل تشغيل الوكيل.
- الخلفية تحفظ رد الوكيل بعد نجاح الطلب أو رسالة الخطأ عند الفشل.
- كل تقرير HTML يرسل للواجهة يحفظ أيضا في Supabase داخل report_artifacts.
- كل تقرير يحصل على report_number يمكن استخدامه لاحقا لاسترجاع التقرير.
- عند بناء سياق النموذج، يعتمد التطبيق على ملخص الجلسة، آخر الرسائل القصيرة، والوصفات الناجحة من الذاكرة بدلا من إرسال تاريخ طويل.

## تقليل التوكن

- المسار السريع يبقى أولوية:

```text
run_query_pattern
```

- عند وجود تطابق قوي في:

```text
agent_tool_recipes
```

ينفذ التطبيق الأداة مباشرة ثم يرسل للنموذج نتيجة مختصرة فقط للتلخيص.

- النتائج الكاملة تحفظ خارج سياق النموذج في:

```text
report_artifacts
```

## مراقبة الأخطاء

أخطاء الأدوات تسجل في:

```text
agent_errors
chat_messages
```

ثم يمكن تحليلها عبر:

```text
get_failing_intents
```

## ملاحظات أمنية

ظهر تحذير Supabase أن بعض الجداول القديمة في schema العام بدون RLS، مثل:

```text
reports
documents
```

هذه الخطة لا تعالج ذلك تلقائيا حتى لا تكسر الوصول الحالي. يجب معالجة ذلك في مهمة منفصلة بسياسات مناسبة.

## تحقق التنفيذ

- فحص الجداول والدوال عبر Supabase MCP.
- تشغيل فحص Rust:

```powershell
cd src-tauri
cargo check
```

- تشغيل بناء الواجهة:

```powershell
npm run build
```

- اختبار عملي:
  - كشف حساب عميل مختصر.
  - كشف حساب عميل مفصل.
  - توليد تقرير ثم طلبه بالرقم.
  - تكرار نفس السؤال والتأكد من استخدام الذاكرة أو الوصفة السريعة.
