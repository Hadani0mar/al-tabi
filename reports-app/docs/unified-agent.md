# الوكيل الموحد الذكي

هذا الملف يوثق التصميم الحالي للوكيل بعد إلغاء فكرة الوضع السريع والوضع المتقدم.

## الهدف

- مسار واحد دائمًا للتطبيق وتليجرام.
- تنفيذ التقارير المعروفة من الأنماط المعتمدة بدون تغيير أي استعلام.
- حفظ التقرير الكامل خارج سياق النموذج.
- تمكين المستخدم من سؤال الوكيل بعد صدور التقرير، مثل:

```text
شن رأيك فيه؟
من أكثر موظف؟
كم إجمالي الدائن؟
هل العميل يدفع؟
ارسل التقرير 1002
```

## المسار الحالي

الوكيل يعمل كمسار واحد باسم داخلي:

```text
UnifiedAgent
```

ويتكون عمليًا من:

```text
Intent Router
Tool Executor
Report-Aware Answer Composer
```

قبل استدعاء النموذج يحاول التطبيق برمجيًا:

- تنفيذ التقرير المعروف مباشرة إذا تطابق الطلب مع نمط موجود.
- استخدام آخر عميل محفوظ عند طلب متابعة مثل:

```text
له
نفس العميل
اعرض مفصل له
```

- الإجابة من آخر تقرير محفوظ إذا كان السؤال عن قيم داخل التقرير.
- استرجاع التقرير برقم واضح عند طلب:

```text
ارسل التقرير 1002
```

## أدوات النموذج

مجموعة أدوات النموذج في المسار الموحد مختصرة:

```text
run_erp_report
run_query_pattern
answer_from_report
get_report_artifact
export_last_result
validate_sql
get_table_sample
get_database_views
get_product_schema
```

أدوات إنشاء أو إرسال ملفات:

```text
PDF
```

ليست ضمن أدوات النموذج في الوكيل الموحد. الطباعة والإرسال تتم برمجيًا من التقرير المحفوظ.

## الذاكرة

الذاكرة المشتركة بين الدسكتوب وتليجرام تعتمد على:

```text
chat_sessions
chat_messages
report_artifacts
agent_tool_recipes
agent_errors
```

تم ترحيل المحادثات القديمة من:

```text
user_chats
```

إلى:

```text
chat_sessions
chat_messages
```

مع إبقاء الجدول القديم بدون حذف.

## تليجرام

جلسة تليجرام تحفظ بصيغة:

```text
telegram:<chat_id>
```

وتستخدم نفس الذاكرة ونفس أرقام التقارير. عند توليد تقرير من تليجرام يرسل البوت ملخصًا عربيًا مختصرًا ورقم التقرير، ولا يرسل جدولًا طويلًا إلا عند طلب الملف أو التقرير الكامل.

## حالة Supabase

الجداول الجديدة مفعّل عليها:

```text
RLS
```

الجداول القديمة التالية ما زالت بدون:

```text
RLS
```

```text
reports
documents
report_categories
report_parameters
execution_logs
```

لم يتم تفعيل السياسات عليها ضمن هذا التغيير حتى لا ينكسر الوصول الحالي. يجب فتح مهمة منفصلة لتأمينها بسياسات مناسبة.
