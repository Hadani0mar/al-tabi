# دليل حل مشكلة الاتصال بقاعدة البيانات

الخطأ "المشين ترفض الاتصال" يعني أن التطبيق لا يصل إلى المنفذ الذي يستمع عليه SQL Server.
نفّذ الأقسام بالترتيب. الأقسام الأولى تشخيصية داخل SSMS، والأخيرة تنفّذ خارج SSMS.

---

## القسم الأول: التشخيص داخل SSMS

افتح SSMS واتصل بالقاعدة محلياً على جهاز العميل، ثم نفّذ كل استعلام وسجّل نتيجته.

### 1. اسم السيرفر والـ Instance

```sql
SELECT
  @@SERVERNAME                              AS ServerName,
  SERVERPROPERTY('MachineName')             AS MachineName,
  SERVERPROPERTY('InstanceName')            AS InstanceName,
  SERVERPROPERTY('IsClustered')             AS IsClustered;
```

إذا ظهر شيء في عمود InstanceName (مثل SQLEXPRESS) فهذا Named Instance ويحتاج معاملة خاصة.
إذا كان InstanceName فارغاً (NULL) فهو Default Instance ويتصل مباشرة بالمنفذ 1433.

### 2. المنفذ الفعلي الذي يستمع عليه SQL Server

```sql
SELECT local_net_address AS ListenIP, local_tcp_port AS Port
FROM sys.dm_exec_connections
WHERE session_id = @@SPID;
```

سجّل رقم المنفذ. إن كان 1433 فهو القياسي. إن كان رقماً آخر (مثل 49718) فهو منفذ ديناميكي.
إن كان local_tcp_port فارغاً فأنت متصل عبر Shared Memory وليس TCP، وهذا هو سبب المشكلة.

### 3. وضع المصادقة

```sql
SELECT CASE SERVERPROPERTY('IsIntegratedSecurityOnly')
  WHEN 1 THEN N'Windows فقط — لن يعمل اسم مستخدم وكلمة سر'
  WHEN 0 THEN N'Mixed Mode — يقبل مستخدم وكلمة سر'
END AS AuthMode;
```

إذا ظهر Windows فقط، يجب تفعيل Mixed Mode (القسم الثاني).

### 4. التأكد من وجود مستخدم الاتصال

```sql
SELECT name, is_disabled, type_desc
FROM sys.server_principals
WHERE type IN ('S','U')
ORDER BY name;
```

ابحث عن اسم المستخدم الذي تستخدمه في التطبيق. إذا لم يوجد، أنشئه (القسم الثالث).

## القسم الثاني: تفعيل Mixed Mode

نفّذ هذا فقط إذا أظهر استعلام المصادقة "Windows فقط".

```sql
USE [master];
GO
EXEC xp_instance_regwrite
  N'HKEY_LOCAL_MACHINE',
  N'Software\Microsoft\MSSQLServer\MSSQLServer',
  N'LoginMode', REG_DWORD, 2;
GO
```

ملاحظة: يتطلب إعادة تشغيل خدمة SQL Server حتى يسري المفعول (القسم الخامس).

---

## القسم الثالث: إنشاء مستخدم الاتصال

نفّذ هذا فقط إذا لم يوجد المستخدم. غيّر الاسم وكلمة السر حسب حاجتك.

```sql
USE [master];
GO
CREATE LOGIN [report_user] WITH PASSWORD = N'ضع_كلمة_سر_قوية_هنا',
  CHECK_POLICY = OFF;
GO
ALTER SERVER ROLE [sysadmin] ADD MEMBER [report_user];
GO
```

إذا كان المستخدم موجوداً لكن نسيت كلمة سره، أعد تعيينها:

```sql
ALTER LOGIN [report_user] WITH PASSWORD = N'كلمة_سر_جديدة';
ALTER LOGIN [report_user] ENABLE;
GO
```

## القسم الرابع: تفعيل TCP/IP وتثبيت المنفذ

هذه الخطوات لا تُنفّذ في SSMS بل عبر PowerShell بصلاحية Administrator.
افتح PowerShell كمسؤول والصق الأوامر.

### تفعيل بروتوكول TCP/IP

```powershell
$smo = 'Microsoft.SqlServer.Management.Smo.'
$wmi = New-Object ($smo + 'Wmi.ManagedComputer')
$instance = $wmi.ServerInstances | Select-Object -First 1 -ExpandProperty Name
$tcp = $wmi.GetSmoObject("ManagedComputer[@Name='$env:COMPUTERNAME']/ServerInstance[@Name='$instance']/ServerProtocol[@Name='Tcp']")
$tcp.IsEnabled = $true
$tcp.Alter()
Write-Output "TCP/IP enabled for instance: $instance"
```

### تثبيت المنفذ على 1433

```powershell
$ipAll = $tcp.IPAddresses | Where-Object { $_.Name -eq 'IPAll' }
$ipAll.IPAddressProperties['TcpDynamicPorts'].Value = ''
$ipAll.IPAddressProperties['TcpPort'].Value = '1433'
$tcp.Alter()
Write-Output "Port fixed to 1433"
```

## القسم الخامس: تشغيل الخدمات وفتح الجدار الناري

### إعادة تشغيل خدمة SQL Server

```powershell
Get-Service | Where-Object { $_.Name -like 'MSSQL*' } | Restart-Service -Force
```

### تشغيل خدمة SQL Browser (مطلوبة للـ Named Instance)

```powershell
Set-Service -Name 'SQLBrowser' -StartupType Automatic
Start-Service -Name 'SQLBrowser'
```

### فتح المنافذ في جدار الحماية

```powershell
New-NetFirewallRule -DisplayName 'SQL Server TCP 1433' -Direction Inbound -Protocol TCP -LocalPort 1433 -Action Allow
New-NetFirewallRule -DisplayName 'SQL Browser UDP 1434' -Direction Inbound -Protocol UDP -LocalPort 1434 -Action Allow
```

---

## القسم السادس: اختبار الاتصال من جهاز التطبيق

من جهاز التطبيق (وليس جهاز السيرفر) افتح PowerShell ونفّذ:

```powershell
Test-NetConnection -ComputerName عنوان_السيرفر -Port 1433
```

إذا ظهر TcpTestSucceeded : True فالمنفذ مفتوح والتطبيق سيتصل.
إذا ظهر False فالمشكلة ما زالت في الجدار الناري أو أن TCP لم يُفعّل بعد.

---

## ملخص سريع لأكثر حالة شائعة

معظم أنظمة الصيدليات تعمل على SQL Express بـ Named Instance. الحل في 90 بالمئة من الحالات:

1. فعّل TCP/IP (القسم الرابع)
2. ثبّت المنفذ على 1433 (القسم الرابع)
3. أعد تشغيل الخدمة وافتح الجدار الناري (القسم الخامس)
4. في التطبيق أدخل عنوان السيرفر والمنفذ 1433

بعد ذلك يجب أن ينجح الاتصال.
